Subscription procedures work over the same typed MessagePort channels as
queries and mutations.

## Defining a Subscription

```typescript
// src/main/router.ts
import { initTRPC } from '@trpc/server';
import { z } from 'zod';

const t = initTRPC.create();

export const appRouter = t.router({
  onTimer: t.procedure
    .input(z.object({ intervalMs: z.number().min(100) }))
    .subscription(async function* ({ input, signal }) {
      let count = 0;
      while (!signal.aborted) {
        // No transformer is configured on `initTRPC.create()`, so values must
        // be JSON-serializable. Emit an ISO string instead of a `Date` object.
        yield { count: count++, timestamp: new Date().toISOString() };
        await new Promise((resolve) => setTimeout(resolve, input.intervalMs));
      }
    }),
});
```

<Aside type="tip">
The `signal` parameter is an `AbortSignal` that is triggered when the client unsubscribes or the port is closed. Always check `signal.aborted` in your loops to clean up properly.
</Aside>

## Subscribing from the Renderer

```typescript
// src/renderer/trpc.ts
import { createElectronTRPCClient } from 'electron-messageport-trpc/renderer';
import type { AppRouter } from '../main/router';

export const trpc = createElectronTRPCClient<AppRouter>();
```

```typescript
// src/renderer/index.ts
import { trpc } from './trpc';

const subscription = trpc.onTimer.subscribe(
  { intervalMs: 1000 },
  {
    onStarted() {
      console.log('Subscription started');
    },
    onData(value) {
      // value is typed as { count: number; timestamp: string }
      console.log(`Tick #${value.count} at ${value.timestamp}`);
    },
    onError(err) {
      console.error('Subscription error:', err);
    },
    onStopped() {
      console.log('Subscription stopped by the server');
    },
    onComplete() {
      console.log('Subscription ended');
    },
  },
);

subscription.unsubscribe();
```

<Aside type="note" title="Using multiple typed channels?">
The client above is the default flat shape: `createElectronTRPCClient<AppRouter>()` with calls like `trpc.onTimer.subscribe(...)`. If you run several routers across channels (for example a main router plus a utility worker), build the client from a `defineElectronTRPC()` registry and namespace by channel: `createElectronTRPCClient(electronTRPC).worker.heartbeats.subscribe(...)`. See the [multi-topology guide](/guides/multi-topology/) for the full setup.
</Aside>

## Error Handling

If the subscription procedure throws an error, the client's `onError` callback receives a `TRPCClientError` instance.

## Lifecycle and Cleanup

The server's async generator keeps running until the client unsubscribes or the channel closes. To stop early or release resources, check `signal.aborted` in your loop and exit when it becomes `true`.

- **Client unsubscribes**: calling `unsubscribe()` stops the subscription; the server's `signal` is aborted so the generator can clean up.
- **Port closes**: all active subscriptions on that channel stop and their signals are aborted.
- **`destroy()` called**: behaves like a port close -- all subscriptions stop and the channel is closed.
