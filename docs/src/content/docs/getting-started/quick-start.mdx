This guide walks through a minimal setup where a **renderer** process calls procedures on the **main** process.

<Steps>

1. **Define your tRPC router (main process)**

    ```typescript
    // src/main/router.ts
    import { initTRPC } from '@trpc/server';
    import { z } from 'zod';

    const t = initTRPC.create();

    export const appRouter = t.router({
      greet: t.procedure
        .input(z.object({ name: z.string() }))
        .query(({ input }) => {
          return `Hello, ${input.name}!`;
        }),
    });

    export type AppRouter = typeof appRouter;
    ```

2. **Attach your router to the window (main process)**

    ```typescript
    // src/main/index.ts
    import { app, BrowserWindow } from 'electron';
    import { createElectronTRPCMain } from 'electron-messageport-trpc/main';
    import { appRouter } from './router';

    app.whenReady().then(() => {
      const win = new BrowserWindow({
        webPreferences: {
          preload: '/path/to/preload.js',
        },
      });

      createElectronTRPCMain({
        router: appRouter,
        windows: [win],
      });

      win.loadFile('index.html');
    });
    ```

3. **Expose the tRPC bridge (preload script)**

    ```typescript
    // src/preload/index.ts
    import { exposeElectronTRPC } from 'electron-messageport-trpc/preload';

    exposeElectronTRPC();
    ```

4. **Create the tRPC client (renderer process)**

    ```typescript
    // src/renderer/index.ts
    import { createElectronTRPCClient } from 'electron-messageport-trpc/renderer';
    import type { AppRouter } from '../main/router';

    const trpc = createElectronTRPCClient<AppRouter>();

    const greeting = await trpc.greet.query({ name: 'World' });
    console.log(greeting); // "Hello, World!"
    ```

</Steps>

## What Just Happened?

1. `createElectronTRPCMain()` connects `appRouter` to each window on the default channel.
2. `exposeElectronTRPC()` receives transferred MessagePorts in preload and exposes the bridge to the renderer.
3. `createElectronTRPCClient<AppRouter>()` creates a normal tRPC client for the default channel. The MessagePort setup stays inside the Electron helpers.

<Aside type="tip">
Use `defineElectronTRPC()` when one renderer needs multiple typed channels, such as `main` and `utility`. See [Multi-Topology](/guides/multi-topology/).
</Aside>

<Aside type="tip">
The low-level exports are still available if you want custom lifecycle control or direct MessagePort access.
</Aside>
