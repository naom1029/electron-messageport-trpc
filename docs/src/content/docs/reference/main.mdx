## `createElectronTRPCMain(opts)`

Connects one or more typed registry channels to renderer windows.

```typescript
import { createElectronTRPCMain } from 'electron-messageport-trpc/main';
import { electronTRPC } from './trpc';
import { appRouter } from './router';

createElectronTRPCMain({
  channels: electronTRPC,
  windows: [win],
  routers: {
    main: appRouter,
  },
});
```

`routers` only accepts keys from the registry. Each router must match the
router type registered for that channel.

---

## `createElectronTRPCUtilityClient(opts)`

Creates a typed tRPC client from the main process to one utility process.

```typescript
const worker = createElectronTRPCUtilityClient({
  channel: electronTRPC.worker,
  utility: child,
});

await worker.render.mutate(input);
```

This wraps `MessageChannelMain`, `utility.postMessage()`, and `mainPortLink()`.

---

## `createElectronTRPCUtilityPool(opts)`

Creates typed clients for multiple utility process instances that expose the
same router type.

```typescript
const workers = createElectronTRPCUtilityPool({
  channel: electronTRPC.worker,
  utilities: {
    a: workerA,
    b: workerB,
  },
});

await workers.get('a').render.mutate(input);
```

The registry channel carries the router type. The pool key chooses the runtime
utility process instance.

---

## `createElectronTRPCRendererUtilityBridge(opts)`

Brokers a renderer-to-utility MessagePort. The main process creates the port
pair and transfers one side to the renderer and the other side to the utility
process.

```typescript
createElectronTRPCRendererUtilityBridge({
  window: win,
  channel: electronTRPC.worker,
  utility: child,
});
```

The main process stays out of the tRPC request path after the port is handed
off.

## `createWindowMessagePortHandler(opts)`

Creates and maintains MessagePort-backed tRPC handlers for one or more Electron windows.

### Parameters

```typescript
interface CreateWindowMessagePortHandlerOptions<TRouter extends AnyRouter> {
  router: TRouter;
  windows: readonly BrowserWindow[];
  createContext?: (opts: { window: BrowserWindow }) => Promise<unknown>;
  transformer?: DataTransformerOptions;
}
```

| Parameter | Type | Required | Description |
|---|---|---|---|
| `router` | `AnyRouter` | Yes | Your tRPC router instance |
| `windows` | `readonly BrowserWindow[]` | Yes | Windows that should receive a renderer-side port |
| `createContext` | `(opts: { window: BrowserWindow }) => Promise<unknown>` | No | Factory invoked for each request with the matching window |
| `transformer` | `DataTransformerOptions` | No | Overrides the router's configured data transformer |

### Returns

A `WindowMessagePortHandler` object.

### `WindowMessagePortHandler`

#### `handler.destroy()`

Stops all managed window handlers and removes their Electron event listeners.

### Example

```typescript
import { createWindowMessagePortHandler } from 'electron-messageport-trpc/main';
import { appRouter } from './router';

app.whenReady().then(() => {
  const win = new BrowserWindow({ /* ... */ });

  createWindowMessagePortHandler({
    router: appRouter,
    windows: [win],
    createContext: async ({ window }) => ({
      windowId: window.id,
    }),
  });
});
```

<Aside type="tip">
For new code, prefer `createElectronTRPCMain()`. This lower-level helper remains available for single-router window wiring.
</Aside>

## `mainPortLink(opts)`

Creates a tRPC client link for `MessagePortMain`-compatible ports in the main process. Use this when the main process is the tRPC client, such as calling a utility process over `MessageChannelMain`.

### Parameters

```typescript
interface MainPortLinkOptions {
  port: MessagePortMain | Promise<MessagePortMain>;
  transformer?: DataTransformerOptions;
}
```

| Parameter | Type | Required | Description |
|---|---|---|---|
| `port` | `MessagePortMain \| Promise<MessagePortMain>` | Yes | The client-side main-process port |
| `transformer` | `DataTransformerOptions` | No | Data transformer matching the remote router |

### Example

```typescript
import { createTRPCClient } from '@trpc/client';
import { MessageChannelMain, utilityProcess } from 'electron';
import { mainPortLink } from 'electron-messageport-trpc/main';

const child = utilityProcess.fork('utility-worker.js');
const { port1, port2 } = new MessageChannelMain();

child.postMessage({ type: 'connect' }, [port1]);

const client = createTRPCClient({
  links: [mainPortLink({ port: port2 })],
});
```

## `createPortBroker()`

Creates a port broker that generates MessagePort pairs for connecting renderers to the main process.

### Returns

A `PortBroker` object.

### `PortBroker`

#### `broker.createRendererPort(webContents)`

Creates a `MessageChannelMain` pair, sends one port to the specified renderer, and returns the other.

**Parameters:**

| Parameter | Type | Description |
|---|---|---|
| `webContents` | `WebContents` | The target renderer's `webContents` object (e.g., `win.webContents`) |

**Returns:**

```typescript
{
  serverPort: MessagePortMain;
}
```

The `serverPort` is the main-process side of the channel. Pass it to `createPortHandler()` to start handling tRPC requests.

### Example

```typescript
import { createPortBroker } from 'electron-messageport-trpc/main';

const broker = createPortBroker();

// After the window loads:
const { serverPort } = broker.createRendererPort(win.webContents);
```

<Aside type="note">
Internally, `createRendererPort` calls `webContents.postMessage()` with the port as a transferable. The preload script must call `exposePortReceiver()` to receive it.
</Aside>

## `createPortHandler(opts)`

Attaches a tRPC request handler to a MessagePort. It listens for incoming tRPC requests (queries, mutations, subscriptions) and routes them to the given router.

<Aside type="caution">
This is a low-level API. The supplied port should be dedicated to the electron-messageport-trpc protocol; do not multiplex app-defined `postMessage()` traffic on the same port. Messages that do not match the protocol are discarded.
</Aside>

### Parameters

```typescript
interface CreatePortHandlerOptions<TRouter extends AnyRouter> {
  port: MessagePortLike;
  router: TRouter;
  createContext?: () => Promise<unknown>;
  transformer?: DataTransformerOptions;
}
```

| Parameter | Type | Required | Description |
|---|---|---|---|
| `port` | `MessagePortMain` | Yes | The server-side MessagePort (from `createRendererPort`) |
| `router` | `AnyRouter` | Yes | Your tRPC router instance |
| `createContext` | `() => Promise<unknown>` | No | Factory function called for every request to create the context object |
| `transformer` | `DataTransformerOptions` | No | Overrides the router's configured data transformer |

### Returns

A `PortHandler` object.

### `PortHandler`

#### `handler.destroy()`

Stops all active subscriptions, aborts their `AbortController`s, and closes the underlying MessagePort.

Call this when the associated `BrowserWindow` is closed or the connection is no longer needed.

### Example

```typescript
import { createPortBroker, createPortHandler } from 'electron-messageport-trpc/main';
import { appRouter } from './router';

const broker = createPortBroker();

app.whenReady().then(() => {
  const win = new BrowserWindow({ /* ... */ });

  win.webContents.on('did-finish-load', () => {
    const { serverPort } = broker.createRendererPort(win.webContents);

    const handler = createPortHandler({
      port: serverPort,
      router: appRouter,
      createContext: async () => ({
        // per-request context
        windowId: win.id,
      }),
    });

    // Clean up when the window closes
    win.on('closed', () => {
      handler.destroy();
    });
  });
});
```

### Subscription Handling

When a `subscription` request arrives:

1. The procedure is called and must return an async iterable.
2. The handler sends `{ kind: 'result', type: 'started' }`.
3. Each yielded value is sent as a `{ kind: 'result', type: 'data' }` message.
4. If the procedure yields `tracked(id, data)`, the handler sends the tracked event ID for tRPC v11 resumption.
5. When the client sends `{ kind: 'subscription.stop' }`, the `AbortController` is aborted.
6. When the port closes, all active subscriptions are aborted automatically.
