This package uses an internal MessagePort protocol between its client links and
server handlers. The message shape is private implementation detail, not a
public API. Use `portLink()`, `mainPortLink()`, `createPortHandler()`, or
the higher-level typed registry helpers instead of posting protocol messages
manually.

<Aside type="note">
Each MessagePort passed to this package should be dedicated to
electron-messageport-trpc traffic. Do not mix application-defined
`postMessage()` traffic on the same port.
</Aside>

## Data Flow

Request inputs, response data, and error shapes are sent through
`MessagePort.postMessage()`.

When a tRPC data transformer is configured, values are transformed before they
are posted to the port and transformed back after they are received:

- client input: `transformer.input.serialize()`
- server input: `transformer.input.deserialize()`
- server result and error shapes: `transformer.output.serialize()`
- client result and error shapes: `transformer.output.deserialize()`

This mirrors the data-transformer behavior users expect from tRPC links, while
still using MessagePort as the underlying transport.

## Structured Clone

After any configured transformer runs, the value must still be accepted by the
platform Structured Clone implementation used by Electron MessagePort.

Values such as JavaScript `Proxy` objects cannot be posted directly. Configure a
tRPC transformer when you need those values converted into clone-safe data before
transport.

## Binary Data

`Uint8Array`, `ArrayBuffer`, typed arrays, and other Structured Clone values use
the platform MessagePort transport directly.

`Blob` values are encoded by this package before `postMessage()` and restored on
the receiving side. This avoids Electron renderer-to-main MessagePort behavior
where Blob-containing messages can arrive as `null` without throwing from
`postMessage()`.

## Compatibility Notes

This page describes transport behavior, not the official tRPC JSON-RPC wire
format. The internal MessagePort message shape may change between releases.
