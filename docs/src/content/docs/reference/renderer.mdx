## `createElectronTRPCClient(channels, opts?)`

Creates typed tRPC clients for every channel defined by `defineElectronTRPC()`.

```typescript
import { createElectronTRPCClient } from 'electron-messageport-trpc/renderer';
import { electronTRPC } from '../main/trpc';

const trpc = createElectronTRPCClient(electronTRPC);

await trpc.main.ping.query();
await trpc.worker.render.mutate(input);
```

### Options

```typescript
interface CreateElectronTRPCClientOptions<TRegistry> {
  transformer?: DataTransformerOptions;
  channels?: Partial<
    Record<keyof TRegistry, { transformer?: DataTransformerOptions }>
  >;
}
```

Use `channels` when different channel routers need different transformers.

## `portLink(opts)`

Creates a tRPC link that communicates over a MessagePort. Use this as a link in your `createTRPCClient` configuration.

### Parameters

```typescript
interface PortLinkOptions {
  port?: MessagePort | Promise<MessagePort>;
  channel?: string;
  transformer?: DataTransformerOptions;
}
```

| Parameter | Type | Required | Description |
|---|---|---|---|
| `port` | `MessagePort \| Promise<MessagePort>` | No | The renderer-side MessagePort, or a promise that resolves to one |
| `channel` | `string` | No | Channel used when `port` is omitted |
| `transformer` | `DataTransformerOptions` | No | Data transformer matching the remote router |

### Returns

`TRPCLink<TRouter>` -- a tRPC link function compatible with `createTRPCClient`.

### Example

```typescript
import { createTRPCClient } from '@trpc/client';
import { portLink, getPort } from 'electron-messageport-trpc/renderer';
import type { AppRouter } from '../main/router';

const trpc = createTRPCClient<AppRouter>({
  links: [
    portLink({ port: getPort({ channel: 'main' }) }),
  ],
});
```

<Aside type="tip">
For the primary API, prefer `createElectronTRPCClient()`. `portLink()` remains available for custom MessagePort wiring.
</Aside>

### How It Works

1. On first use, the link resolves the port promise and sets up a `message` event listener.
2. Each request is assigned a unique numeric ID.
3. Requests are sent as `ClientMessage` objects via `port.postMessage()`.
4. Responses are matched by ID and dispatched to the appropriate observer.

## `getPort(opts?)`

Returns a `Promise<MessagePort>` that resolves to the port exposed by the preload script.

### Parameters

```typescript
interface GetPortOptions {
  channel?: string;
}
```

### Returns

`Promise<MessagePort>`

### Throws

Throws an `Error` if `exposePortReceiver()` was not called in the preload script (i.e., `window.electronTRPCPort` is `undefined`).

```
Error: electronTRPCPort not found. Did you call exposePortReceiver() in your preload script?
```

### Example

```typescript
import { getPort } from 'electron-messageport-trpc/renderer';

const port = await getPort();
const workerPort = await getPort({ channel: 'worker' });
// Use the port directly, or pass it to portLink
```
